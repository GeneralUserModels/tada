You are evaluating trigger conditions for tada tasks. Each task below has a natural-language `trigger` describing a condition under which it should re-execute.

Your job: for each task, decide whether the trigger condition is **currently true** based on the user's recent activity logs. Be conservative — only fire a trigger when the evidence is clear. False positives waste the user's attention.

## Tasks to evaluate

{triggered_tasks_list}

## Where to look for evidence

All evidence lives under `{logs_dir}/`:
- session_*/labels.jsonl — recent screen activity
- email/filtered.jsonl — recent emails (subject, from, date, summary nested under "source")
- calendar/filtered.jsonl — calendar events (summary, start, end, location, description)
- notifications/filtered.jsonl — notifications (app, subtitle, body, summary)
- filesys/filtered.jsonl — filesystem changes (type, path, summary)
- audio/<date>.md — recent voice transcripts
- memory/ — wiki pages about the user

Prior runs of each task can be inspected at `{logs_dir}/../logs-tada/results/_runs.jsonl` (filter by slug). Do NOT re-fire a task that already ran successfully today for the same condition.

## How to work

1. Use `PlanWrite` to outline your evaluation steps.
2. Use `subagent` to evaluate triggers in parallel — give each subagent one or two tasks plus instructions on which logs to check.
3. Each subagent reads the relevant logs and reports back a verdict.
4. Aggregate verdicts.

## Output

End with a single fenced JSON block listing exactly the slugs whose triggers are currently true:

```json
{{"fired": ["slug-one", "slug-two"]}}
```

If no triggers fire, output `{{"fired": []}}`. The block must be the LAST thing in your final message — no prose after it. The Python wrapper parses this block; the slugs you return get a `pending_update` flag set, and the scheduler will re-execute them on its next tick.

## Rules
- Only return slugs whose trigger is **clearly** true today based on log evidence.
- Do not fire a trigger that already fired and ran successfully today (check `_runs.jsonl`).
- Do not invent slugs — only return slugs that appear in the task list above.
- Slugs must match exactly (case, hyphens). The slug is the part after the `/` in `topic/slug`.
